Where this deliverable stops · what’s next
The data is now model-ready — the modelling ladder is the next step
This hand-off ends at the three nested feature sets — M0 location + season → M1 + weather → M2 + LST. Each rung adds exactly one lever, so the next stage can measure what each lever is worth on its own. The road ahead, in four moves:
  1. 1Build the models. Two trivial baselines (predict the mean · “same as yesterday”), then RandomForest and XGBoost on each feature set — so any gain has a floor it has to beat.
  2. 2Grade them honestly. A model can look clever by memorising instead of learning — so we test it like fair exams. A new place: hide a whole sensor during training, then test on it — can it handle a neighbourhood it has never seen? The future: train on the early months and predict the later ones, never peeking at the days in between. The easy random exam lets it peek at near-identical neighbours; these two do not — and whatever score it loses on the hard exams was leakage (looking smart by memorising, not predicting).
  3. 3Run the ladder. Climb M0 → +weather → +LST one rung at a time and read each rung’s isolated jump — the clean test of which lever actually moves PM2.5.
  4. 4Turn it toward forecasting. Move from “what is the air now?” to “what will it be tomorrow?” — the usable smart-city product, and where the work heads after the ladder.
